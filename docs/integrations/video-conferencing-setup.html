1 · How the integration works

When an agent clicks Start Bridge Call on an incident, the Helpdesk server does this:

  1. Reads the Active provider from Settings → Integrations → Video Bridge.
  2. Loads the credentials saved for that provider.
  3. Calls the provider's API to create a real meeting.
  4. Stamps the join URL onto the incident and notifies the on-call team.
Tip Only one provider is active at a time. The system stores credentials for all four simultaneously, so you can switch providers without re-entering anything.

Provider implementations live in server/src/lib/bridge/. The Test Connection button creates and immediately deletes a real meeting — the only reliable way to verify credentials end-to-end.

2 · Where to enter credentials

  1. Sign in to Helpdesk as an admin.
  2. Open Settings → Integrations.
  3. Scroll to Video Bridge (Incident Bridge Calls).
  4. Pick a provider from the Active provider dropdown.
  5. Fill in the fields that appear, click Save.
  6. Click Test Connection. ✓ Connection successful = done.

3 · Google Meet

G
Google Meet
OAuth 2.0 + long-lived Refresh Token · Setup ~15 min · Tokens never expire if consent screen is Internal or Published

What you need

FieldSource
OAuth Client IDGoogle Cloud Console → Credentials → OAuth 2.0 Client IDs
OAuth Client SecretSame credential entry
Refresh TokenGenerated via OAuth Playground

3.1 Create a Google Cloud project

  1. Open console.cloud.google.com.
  2. Top-left project picker → New Project.
  3. Name: Zentra Video Meet. If the Organization dropdown shows your Workspace domain, keep it selected.
  4. Create and wait ~10 s.

3.2 Enable the Calendar API

From the project, open APIs & Services → Library, search for Google Calendar API, click Enable. Optionally also enable Google Meet API for richer metadata.

Note Skipping this step is the #1 cause of 403 accessNotConfigured when you click Test Connection.

3.3 Configure the OAuth consent screen

  1. Open APIs & Services → OAuth consent screen.
  2. User type: Internal (if on Workspace) or External (personal Gmail).
  3. Fill in App name, support email, developer email → Save and continue.
  4. Skip Scopes → continue.
  5. External only: add the authorizing user under Test users.
  6. If status is "Testing" and user type is External, click Publish app — this is what gives the refresh token an indefinite lifetime.

3.4 Create the OAuth client

  1. APIs & Services → Credentials → + Create Credentials → OAuth client ID.
  2. Application type: must be Web application (other types can't use the refresh-token flow).
  3. Name: Zentra Helpdesk Bridge.
  4. Authorized JavaScript origins: 
    https://developers.google.com
  5. Authorized redirect URIs: 
    https://developers.google.com/oauthplayground
  6. Click Create. Copy Client ID and Client Secret from the dialog — the secret is shown only at creation.

3.5 Generate a refresh token via OAuth Playground

  1. Open developers.google.com/oauthplayground.
  2. Click the ⚙ gear (top right) → tick Use your own OAuth credentials → paste your Client ID and Secret → Close.
  3. Step 1 — in the Input your own scopes box paste: 
    https://www.googleapis.com/auth/calendar.events
    Click Authorize APIs, sign in, click Allow.
  4. Step 2 — click Exchange authorization code for tokens. Copy the Refresh token (starts 1//…).

4 · Microsoft Teams

T
Microsoft Teams
Azure AD App Registration with application permissions · Setup ~30 min · Requires Global Admin consent + Teams policy grant

What you need

FieldSource
Tenant IDAzure AD Overview
Client IDApp Registration Overview
Client SecretApp Registration → Certificates & secrets
Organizer User IDUPN (email) or Object ID of the meeting organizer

4.1 Register the app

  1. Open portal.azure.com as a tenant admin.
  2. Search Azure Active Directory (now Microsoft Entra ID).
  3. App registrations → + New registration.
  4. Name: Zentra Helpdesk Bridge.
  5. Supported account types: Single tenant.
  6. Redirect URI: leave blank.
  7. Register.

4.2 Copy Tenant ID and Client ID

On the Overview page: copy Application (client) ID and Directory (tenant) ID.

4.3 Create a Client Secret

  1. Certificates & secrets → + New client secret.
  2. Expires: 24 months (or per policy).
  3. Add.
  4. Copy the Value column immediately — shown once only.
Critical If you don't copy the secret value at this exact moment, delete the secret and create a new one. Azure will not show it again.

4.4 Grant API permissions

  1. API permissions → + Add a permission → Microsoft Graph → Application permissions (not Delegated).
  2. Tick OnlineMeetings.ReadWrite.All → Add permissions.
  3. Click Grant admin consent for <tenant> — needs Global Admin.

4.5 Authorize the app to impersonate the organizer

Teams meeting creation via application permissions also requires an application access policy:

Install-Module MicrosoftTeams -Scope CurrentUser
Connect-MicrosoftTeams
New-CsApplicationAccessPolicy -Identity Zentra-Bridge -AppIds "<APP_ID>" -Description "Allow Zentra to schedule meetings on behalf of users"
Grant-CsApplicationAccessPolicy -PolicyName Zentra-Bridge -Identity "<UPN>"

Wait ~30 min for the policy to propagate.

5 · Zoom

Z
Zoom
Server-to-Server OAuth · Setup ~5 min · No user consent, no expiry

What you need

FieldSource

Client IDSame page

5.1 Create the Server-to-Server OAuth app

  1. Open marketplace.zoom.us, sign in.
  2. Top-right Develop → Build App.
  3. Pick Server-to-Server OAuth → Create.
  4. Name: Zentra Helpdesk Bridge.

5.2 Copy credentials

On App Credentials: copy Account ID, Client ID, Client Secret.

5.3 Configure scopes

  1. Information: fill mandatory fields.
  2. Scopes → add:
    • meeting:write:admin — create meetings
    • meeting:read:admin — read meetings (Test Connection uses this)
  3. Continue → Activation → Activate your app.

6 · Cisco Webex

W
Cisco Webex
Bot Token · Setup ~3 min · No OAuth, no expiry (bot tokens are permanent)

What you need

FieldSource
Bot Tokendeveloper.webex.com → your bot
Site URLYour Webex domain, e.g. company.webex.com

6.1 Create a bot

  1. Open developer.webex.com/my-apps, sign in.
  2. Create a New App → Bot.
  3. Bot name: Zentra Helpdesk; bot username must be globally unique, e.g. zentra-helpdesk@webex.bot.
  4. Add Bot.
  5. Copy the Bot Access Token — shown once. Regenerate via My Apps → bot → Regenerate access token if lost.
Production note Use a bot token in production — permanent. Personal Access Tokens from developer.webex.com expire after 12 h and are fine only for short-lived testing.

6.2 Site URL

The domain you sign in to, e.g. company.webex.com — no https:// and no path.

8 · Troubleshooting cheat sheet

  1. [none] No video bridge provider is configured — Active provider is still set to None.
  2. Incomplete <provider> configuration — one of the required fields is blank.
  3. Test passes but agents see no link — check incident detail page; check server logs for the provider-specific error.
  4. Worked yesterday, fails today — rotated Google secret, expired Webex PAT (12 h), or revoked OAuth consent. Bot tokens, refreshed Google tokens (Published), and Zoom S2S do not expire.
Pro tip When Test Connection fails, server logs print the exact provider response. Open server stdout (or your hosting log viewer) and look for lines starting with [googlemeet], [teams], [zoom], [webex]. The error is always more specific in the log than in the toast.
